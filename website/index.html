v2.86.0 — Phases 230–246

CEREBRUM knows
what it doesn't know.
And acts on it.

Most AI systems produce answers. CEREBRUM v2.86.0 produces answers and a structured model of its own epistemic state — then uses that model to automatically retry uncertain queries, resolve contradictions via source credibility, and self-correct scoring parameters without human feedback.

🎯

Confidence is a number, not a feeling

Score margin between top answer and runner-up. Beam score variance (Beta distribution). Branch count. Three independent signals quantify exactly how decisive the answer is.

🔍

It knows where its knowledge came from

PubMed edges score at 0.95 credibility. Synthetic inferences score at 0.30. Source credibility flows directly into path scores — authoritative evidence wins on merit, not position.

⚡

It knows which signal drove the answer

Every response identifies the dominant CSA feature: community structure, semantic similarity, source credibility, temporal freshness, or hub authority. No black box.

⚠️

It detects its own knowledge gaps

When no paths converge above threshold, the system flags a knowledge gap rather than hallucinating a confident wrong answer. "I don't know" is a valid — and surfaced — result.

🔬

Causal vs. correlational paths

CausalDiscoveryEngine automatically classifies relation types as causal or correlational. Every answer reports what fraction of its reasoning chain is causal — without manual annotation.

🧠

Second-order reasoning about reasoning

Meta-relation traversal lets CEREBRUM reason about the structure of its own knowledge graph: "What kinds of connections link disease to gene?" — answered by traversing relation types, not entities.

🔁

Uncertainty triggers automatic retry

Phase 221: when epistemic uncertainty exceeds the Bayesian default variance and a knowledge gap is detected, CEREBRUM automatically widens the beam and expands seeds — no human prompt required. Max one retry per query.

📐

Calibrated confidence, not raw scores

Phase 222: PlattCalibration maps raw beam scores to calibrated probabilities. Expected Calibration Error (ECE) is tracked continuously; if drift exceeds 5%, the system refits automatically from the feedback buffer.

⚡

Self-supervised parameter correction

Phase 223: when gap recovery finds an answer the standard beam missed, that recovered path becomes a self-generated training signal — positive reward on the recovered path, negative reward on the overconfident failure. No POST /feedback needed.

Every /query response now includes:

{
  "answer": "ibuprofen",
  "self_awareness": {
    "answer_confidence":  0.74,        // score margin over runner-up

    "corroboration":     4,           // independent paths that agreed
    "dominant_signal":   "community_structure",
    "causal_fraction":   0.67,        // 2/3 path edges are causal

From Data to Answer

The full CEREBRUM pipeline

Every benchmark number — 60.6%, 83.2%, 66.8%, 95.7% on Hetionet — is the result of this pipeline running end-to-end on whatever graph you load. No training. No LLM. Just graph math.

All benchmark numbers are post-ingestion. The ingestion pipeline — community detection, embedding, GraphProfiler, ParameterInitializer — is what enables high accuracy. BFS without ingestion achieves ~5% H@1. With the full pipeline, zero-config reaches 83.2% H@1 on 1-hop and 56.8% on 3-hop. Running the CMA-ES tuner after ingestion pushes 3-hop to 60.6% (Phase 225-227, full run) with random embeddings, 66.8% (Phase 213, 500-sample) with sentence-transformers.

Build

Graph profiling & self-calibration

CEREBRUM analyzes the graph topology with GraphProfiler, runs TSC community detection (Q=0.88 vs 0.48 for Leiden), loads optional sentence-transformer embeddings, then uses ParameterInitializer to derive all 9 scoring parameters analytically from the graph's own statistics. No tuning required.

3
Query

Cross-Domain Transfer

Load a new graph. No retraining.

Every supervised KGQA system in the comparison table above was trained on MetaQA's movie KG. CEREBRUM generalizes to any knowledge graph with zero retraining. Here it is on Hetionet — a 47,031-entity / 2.25M-edge biomedical graph it had never seen.

Hetionet Biomedical KG

47,031 entities · 2,250,197 edges · 6 query templates · Phase 207–211

Hop H@1 (random) H@1 (sentence) Template
1-hop 95.7% 95.3% disease→gene, gene→pathway, compound→disease
2-hop 47.9% 53.0% disease→gene→pathway (+10pp with sentence)
3-hop 79.5% 49.2% disease→compound→gene (cross-type ceiling†)

† Cross-type 3-hop regression: cosine similarity suppresses valid disease→gene→compound paths because entity types are maximally semantically distant. Random embeddings have no such bias. Documented known limitation — not addressable by parameter tuning.

0.8% → 95.7%

Baseline BFS (no reasoning priors) scores 0.8% on the disease-associates-gene template. CEREBRUM's Terminal Relation Boost identifies the target relation type from the question text — no annotation required — and the beam focuses immediately on the right edge type. Result: 95.7% H@1 on the same template.

2-hop with sentence: +10pp vs random

On the disease→gene→pathway template, sentence-transformer embeddings genuinely help once properly calibrated: 81.1% vs 71.2% with random. The semantic signal helps the beam identify the correct gene intermediate node type.

Honest about the ceiling

Cross-type 3-hop paths (disease→compound) involve entities with maximally dissimilar semantic types. Training-free cosine similarity penalizes these paths at every hop. This gap requires supervised per-relation-type weight calibration to close — outside the scope of training-free CEREBRUM and documented as such.

None of the supervised MetaQA systems (UniKGQA, NSM, EmbedKGQA, GNN-QE) have published Hetionet results. CEREBRUM achieved these numbers on a domain it was never configured for — zero biomedical labels, zero relation-name hardcoding, zero retraining.

Research

76 documented novel contributions

Each contribution has a documented novelty statement and prior-art analysis. Seven headline algorithms are featured below — the full catalog (76 claims, Phases 1–239) is in the IP documentation.

Community-Structured Attention (CSA)

CORTEX

10-parameter scoring formula using graph community topology as discrete attention heads — the structural analogue of Transformer attention, requiring zero training. First attention formula to include global community co-membership as a term.

Dual-Signal Community Fusion (DSCF)

CORTEX

First community detection algorithm that applies all structural signals to every node simultaneously — not sequentially or on disjoint subsets. Produces communities with richer multi-signal character than any prior method.

Schema-Derived Relation Boost (SDRB)

CORTEX

Derives per-relation scoring weights analytically from the graph's own triple statistics. Eliminates KB-specific tuning entirely. fANOVA confirmed branch_bonus as the dominant scoring signal (46.2% variance).

boost(r) = γ × fan_out(r)^β

ParameterInitializer (Zero-Config)

CORTEX

Maps all 9 scoring hyperparameters to measurable graph statistics via IDF theory, Bayesian evidence combination, and Newman-Girvan modularity. A 2D calibration table (regime × embedding method) covers any knowledge base out-of-the-box. Validated: 56.8% H@1 3-hop with zero tuning.

Experience-Dependent Graph Plasticity

REM Engine

Relay nodes form automatically on frequently-traversed inter-community paths, mimicking synaptic long-term potentiation. First KG system where the graph itself strengthens the paths that work — no external agent required.

Training-Free Abductive Reasoning

HypothesisEngine

Given an effect, infer the most plausible cause via multi-path reverse traversal with Noisy-OR path fusion — entirely without training data. Prior abductive KG work requires trained neural models; CEREBRUM uses graph structure alone.

PathSchemaIndex — Predictive Schema Channel

CORTEX

First predictive reasoning signal in CEREBRUM — predicts the most likely (r₁, r₂) 2-hop relation path before any traversal, by matching question embeddings to graph schema embeddings. Parallel targeted traversals add high-precision candidates the beam may have pruned. Phase 236: +3.5pp H@1 on WebQSP (6.0% → 9.5%).
