\texttt{CEREBRUM\_BEAM\_WIDTH} & No & \texttt{10} & Default beam width \\ \hline

\subsection{Docker}

\subsubsection{Single Container}

\begin{lstlisting}
# Dockerfile (already in repo root)
FROM python:3.11-slim
WORKDIR /app
COPY . .
RUN pip install -e ".[api]"
EXPOSE 8200
CMD ["uvicorn", "api.server:app", "--host", "0.0.0.0", "--port", "8200", "--workers", "4"]
\end{lstlisting}

\begin{lstlisting}
docker build -t cerebrum:1.1.0 .
docker run -d \
  -p 8200:8200 \
  -e CEREBRUM_JWT_SECRET="$(openssl rand -hex 32)" \
  -e CEREBRUM_GRAPH_CSV=/data/my_graph.csv \
  -v /path/to/data:/data \
  --name cerebrum \
  cerebrum:1.1.0
\end{lstlisting}

\subsubsection{Docker Compose (with Reasoning Studio)}

\begin{lstlisting}
# docker-compose.yml
version: "3.9"

services:
  api:
    build: .
    image: cerebrum:1.1.0
    ports:
      - "8200:8200"
    environment:
      CEREBRUM_JWT_SECRET: ${CEREBRUM_JWT_SECRET}
      CEREBRUM_GRAPH_CSV: /data/graph.csv
      CEREBRUM_DB_PATH: /data/cerebrum.db
      CEREBRUM_WORKERS: "4"
    volumes:
      - graph_data:/data
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:8200/health"]
      interval: 30s
      timeout: 10s
      retries: 3

  studio:
    build:
      context: .
      dockerfile: ui/Dockerfile
    ports:
      - "7860:7860"
    environment:
      CEREBRUM_API_URL: http://api:8200
      CEREBRUM_JWT_SECRET: ${CEREBRUM_JWT_SECRET}
    depends_on:
      api:
        condition: service_healthy
    restart: unless-stopped

volumes:
  graph_data:
\end{lstlisting}

\subsection{Reverse Proxy (nginx)}

\begin{lstlisting}
# /etc/nginx/sites-available/cerebrum
server {
    listen 443 ssl http2;
    server_name cerebrum.yourdomain.com;

    ssl_certificate     /etc/letsencrypt/live/cerebrum.yourdomain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/cerebrum.yourdomain.com/privkey.pem;
    ssl_protocols       TLSv1.2 TLSv1.3;

    # API
    location /api/ {
        proxy_pass         http://127.0.0.1:8200/;
        proxy_set_header   Host $host;
        proxy_set_header   X-Real-IP $remote_addr;
        proxy_set_header   X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_read_timeout 30s;

        # Rate limiting
        limit_req zone=cerebrum_api burst=20 nodelay;
    }

    # SSE streams (no timeout)
    location /api/stream/ {
        proxy_pass             http://127.0.0.1:8200/stream/;
        proxy_set_header       Connection '';
        proxy_http_version     1.1;
        chunked_transfer_encoding on;
        proxy_buffering        off;
        proxy_cache            off;
        proxy_read_timeout     3600s;
    }

    # Reasoning Studio
    location / {
        proxy_pass http://127.0.0.1:7860/;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
}

# Rate limiting zone
limit_req_zone $binary_remote_addr zone=cerebrum_api:10m rate=100r/m;
\end{lstlisting}

\subsection{Production Checklist}

\subsubsection{Security}
\begin{itemize}
\item [ ] \texttt{CEREBRUM\_JWT\_SECRET} is a random 256-bit hex string (not the dev default)
\item [ ] \texttt{CEREBRUM\_ALLOW\_ANONYMOUS} is \texttt{false} (default)
\item [ ] TLS 1.2+ enabled at the reverse proxy layer
\item [ ] JWT token TTL set to $\le$ 3600s for high-security envir\-onments
\item [ ] \texttt{CEREBRUM\_AUDIT\_LOG=true} if compliance logging is required

\end{itemize}
\subsubsection{Performance}
\begin{itemize}
\item [ ] \texttt{CEREBRUM\_WORKERS} set to \texttt{2 × CPU\_cores + 1} for CPU-bound workloads
\item [ ] \texttt{CEREBRUM\_BEAM\_WIDTH} tuned for your graph size (see \texttt{docs/PERFORMANCE\_TUNING.md})
\item [ ] SQLite persistence file on SSD storage
\item [ ] For graphs \textgreater  500K edges: consider in-memory adapter (disable SQLite persistence)

\end{itemize}
\subsubsection{Reliability}
\begin{itemize}
\item [ ] Health check endpoint monitored: \texttt{GET /health}
\item [ ] Docker restart policy: \texttt{unless-stopped}
\item [ ] Log rotation configured for uvicorn access logs
\item [ ] Data volume backed up (SQLite DB + graph CSV)

\end{itemize}

\subsection{Monitoring}

The \texttt{/admin/resource-stats} endpoint (requires \texttt{admin} scope JWT) returns current CPU, memory, active query count, and rebalance metrics. Integrate with Prometheus via a scrape config targeting this endpoint.

Key metrics to alert on:
\begin{itemize}
\item \texttt{active\_queries} \textgreater  50 — consider scaling
\item \texttt{queued\_events} \textgreater  5,000 — stream ingest falling behind
\item \texttt{rebalance\_count} growing rapidly — graph instability, check \texttt{q\_drift\_threshold}

\end{itemize}
